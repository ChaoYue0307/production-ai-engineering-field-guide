Learning sequence

From one model call to a production AI system

The curriculum is ordered so each module adds one production capability. Start with a deterministic model harness, then add context, caches, tools, retrieval, evals, observability, safety, and finally cross-stack tradeoff decisions.

  1. Foundation: harnesses, typed requests, traces, and tests.
  2. Input control: context packing, caching, structured outputs, and tools.
  3. System behavior: agents, RAG, evals, observability, and cost.
  4. Scale and safety: inference, tenancy, routing, fallback, and degraded UX.

03Caching

Prompt vs. Semantic Caching

Lower latency and cost without returning stale or cross-tenant data.

  • Prompt caching is exact and safer for stable prefixes.
  • Semantic caching catches paraphrases but carries correctness risk.
  • Safe keys include tenant, model, policy, and freshness dimensions.

04Serving

Inference Stack

Optimize prefill, decode, KV cache memory, throughput, and model compression differently.

  • Prefill scales with prompt length; decode is sequential.
  • KV cache pressure grows with layers, hidden size, context, and batch.
  • Compare batching, paged attention, speculative decoding, quantization, and distillation.

05Contracts

Structured Outputs

Treat JSON output as a contract with validation, repair loops, fallback, and explicit failure states.

  • Catch prose-wrapped JSON, missing fields, wrong types, and invalid enums.
  • Validate outside the model before using data.
  • Return safe failure states when repair does not work.

Function Calling Reliability

Make tool use reliable with narrow contracts, runtime argument validation, and idempotency.

Agent Guardrails And Budgets

Bound agent loops with step budgets, tool budgets, token budgets, and termination conditions.

Retrieval-Augmented Generation

Engineer retrieval, ranking, freshness, grounding, and attribution deliberately.

  • Chunk documents into useful evidence units.

09Evals

Golden Sets And Regression Evals

Treat evals as production infrastructure that catches silent regressions.

  • Use golden, regression, adversarial, judge, and human evals.
  • Track task success, groundedness, JSON validity, tool accuracy, safety, cost, and latency.

10Ops

Observability And Cost

Record traces, spans, tokens, latency, errors, drift signals, and cost attribution.

  • Attribute cost by feature, workflow, tenant, and user journey.
  • Monitor validation failures, output length, fallback rate, and eval scores.
  • Debug failures with trace spans instead of guesses.

11Safety

Safety And Tenancy

Preserve permission boundaries when the model is persuasive, confused, or attacked.

  • Treat retrieved documents as data, not instructions.
  • Enforce authorization outside the model.
  • Protect retrieval, caches, traces, tools, and generated context from cross-tenant leakage.
python labs/10_safety_tenancy/run.py
12Tradeoffs

Production Tradeoffs

Choose between fine-tuning, in-context learning, RAG, distillation, quantization, routing, and fallback.

  • Fine-tuning is wrong for frequently changing facts and permissions.
  • RAG is wrong when retrieval quality is poor or behavior must be learned.
  • Fallback models must preserve required capabilities.

Quality bar

Know when you actually learned it

A topic is not mastered when the page feels familiar. It is mastered when you can implement, break, measure, and explain the behavior under production constraints.

Explain

Give a two-minute explanation of the concept, the failure mode it prevents, and the tradeoff it introduces.

Implement

Point to the file or function in `src/ai_engineering_basics/` that implements the pattern.

Break

Change a budget, schema, cache key, chunk size, route rule, or safety policy so the system fails visibly.

Measure

Add a trace, eval case, cost event, or pytest assertion that catches the failure before users do.

Decide

Say when the technique is the wrong tool, such as fine-tuning for fresh facts or semantic caching for private data.

Recover

Design the fallback: retry, repair, route down, ask for clarification, degrade gracefully, or stop safely.
